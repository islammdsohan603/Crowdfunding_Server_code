## 📖 Overview

**Momentum Server** is the backend for a crowdfunding platform that allows users
to create fundraising campaigns, contribute donations to campaigns, and process
payments securely. It's built as a RESTful API, designed to integrate seamlessly
with any frontend application (React, Next.js, etc.).

---

## ✨ Key Features

- 🔐 **Secure Authentication** — JWT (JSON Web Token) based user authentication
  and authorization
- 📢 **Campaign Management** — Create, update, delete, and list fundraising
  campaigns
- 💳 **Payment Integration** — Secure and reliable donation/payment processing
  powered by Stripe
- 🗄️ **MongoDB Database** — Fast and scalable data storage using the native
  MongoDB driver
- 🌐 **CORS Enabled** — Cross-origin requests supported, so any frontend can
  call the API
- ⚙️ **Environment-based Config** — Sensitive data safely managed with `dotenv`
- ☁️ **Serverless Deployment** — Deployed on Vercel for speed and reliability

## 🚀 Getting Started (Local Setup)

### Prerequisites

- [Node.js](https://nodejs.org/) (v18 or higher recommended)
- A [MongoDB](https://www.mongodb.com/atlas) database (local or Atlas cluster)
- A [Stripe](https://dashboard.stripe.com/) account and API key

### Installation

1. **Clone the repository**

   ```bash
   git clone https://github.com/islammdsohan603/Crowdfunding_Server_code.git
   cd Crowdfunding_Server_code
   ```

2. **Install dependencies**

   ```bash
   npm install
   ```

3. **Create a `.env` file**

   Create a `.env` file in the project root and add the following variables:

   ```env
   PORT=5000
   MONGODB_URI=your_mongodb_connection_string
   JWT_SECRET=your_jwt_secret_key
   STRIPE_SECRET_KEY=your_stripe_secret_key
   ```

4. **Start the server**

   ```bash
   # Development mode (auto-restart with nodemon)
   npx nodemon api/index.js

   # Or run it normally
   node api/index.js
   ```

   By default, the server will run on `http://localhost:5000` (or whichever port
   you set in `.env`).

## 🔑 Environment Variables

| Variable            | Description                              |
| ------------------- | ---------------------------------------- |
| `PORT`              | Port on which the server runs            |
| `MONGODB_URI`       | Your MongoDB connection string           |
| `JWT_SECRET`        | Secret key used to sign JWT tokens       |
| `STRIPE_SECRET_KEY` | Secret key for Stripe payment processing |

> ⚠️ **Note:** Never push your `.env` file to GitHub. Keep it listed in
> `.gitignore`.

## 📡 API Endpoints

> The table below follows a common endpoint pattern for a crowdfunding API.
> Please update it to match the exact routes defined in your `api/index.js`
> file.

| Method   | Endpoint                     | Description                         | Auth Required |
| -------- | ---------------------------- | ----------------------------------- | ------------- |
| `GET`    | `/`                          | Server health/status check          | ❌            |
| `POST`   | `/api/auth/register`         | Register a new user                 | ❌            |
| `POST`   | `/api/auth/login`            | Log in a user (returns a JWT token) | ❌            |
| `GET`    | `/api/campaigns`             | Get a list of all campaigns         | ❌            |
| `GET`    | `/api/campaigns/:id`         | Get details of a specific campaign  | ❌            |
| `POST`   | `/api/campaigns`             | Create a new campaign               | ✅            |
| `PATCH`  | `/api/campaigns/:id`         | Update a campaign                   | ✅            |
| `DELETE` | `/api/campaigns/:id`         | Delete a campaign                   | ✅            |
| `POST`   | `/api/payment/create-intent` | Create a Stripe payment intent      | ✅            |

## 🌍 Live Deployment

This backend server is deployed as a serverless function on **Vercel**:

🔗
**[https://servercode-kappa.vercel.app/](https://servercode-kappa.vercel.app/)**

To deploy on Vercel:

1. Import this project into your Vercel account
2. Add the environment variables (`MONGODB_URI`, `JWT_SECRET`,
   `STRIPE_SECRET_KEY`) in the Vercel Dashboard
3. Click Deploy — based on the `vercel.json` configuration, `api/index.js` will
   automatically be deployed as a serverless function
